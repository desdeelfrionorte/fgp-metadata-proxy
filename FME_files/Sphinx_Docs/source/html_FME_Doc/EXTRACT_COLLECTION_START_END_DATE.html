Description:

Custom transformer utilisé pour extraire une étendue temporelle lorsque possible dans une chaine de caractère. Le custom a 3 actions possibles à poser.

  1. Si la chaine de caractère est vide, on attribue la date de début à 0001-01-01. La date de fin est vide
  2. Si la chaine de caractère match une des expression régulière définies dans le code, le programme extrait la date de début et aussi de fin lorque présente
  3. Si la chaine de caractère ne matche pas l'expression régulière, il est possibile de définir manuellement les valeurs de début/fin grâce à une lookup table

Dans le cas où aucune des trois actions n'est possible, la feature ressort par le port unworking_extraction. Dans ce cas, on doit mettre à jour la lookup table ou bien ajouter une regex dans le code python.

Définifition de la lookup table:

On utilise une lookup table selon template yaml suivant:

CSV_COLUMNS:

source_date_extent: [trim, no_null]

data_collection_start_date: [trim, no_null]

data_collection_end_date: [trim]

CREATE_KEY_VALUE:

NO_DUPLICATE: DUP_1: [source_date_extent]

CHECK_DOMAIN:

Output:

Lorsque l'extraction fonctionne, on ajoute les attributs data_collection_start_date et data_collection_end_date. Lorsque ça ne fonctionne pas, l'attribut unmatched_date_extent contient la chaine de caractère non matchée.
